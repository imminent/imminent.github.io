Library

Butterknife (on Maven Central) version 1.2.0 at time of writting

Purpose

Replacement for Activity.findViewById(int) and View.findViewById(int).

Advantage

The first point is that there is nothing magical about butterknife. It simply generates the code like normal, and provides a method for use when you want to grab the View's. The next point is that it provides a pretty semantic annotation-based declaration of the View's to grab. Call it annotated View injection, which is technically different than dependency injection. Its just finding the reference to a View object that's current instantiated in the current Context.

{% highlight java %} public class ExampleActivity extends Activity { @InjectView(R.id.title) /* package */TextView title; @InjectView(R.id.go) /* package */Button go; @InjectView(R.id.toggle) /* package */CheckBox toggle; @Override public void onCreate(Bundle icicle) { super.onCreate(icicle); setContentView(R.layout.simple_activity); Views.inject(this); //TODO: User "injected" views… } } {% endhighlight %}

Where Views.inject(this); executes the usual findViewById commands for the "injected" View's. And you can see how it cleans up the onCreate code, and makes increases readability of the goal of retrieving the Views for use in the class. This injection can be used in Fragment and View classes, and arbitrary classes that are given an Activity or View.

This library is pretty well-documented, and explains its capabilities well.
